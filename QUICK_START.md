# الخطوات المفصلة لكل عضو - من البداية للنهاية

## 🚀 الخطوات التي يقوم بها كل عضو في الفريق

---

### 🔹 الخطوة 1: تنزيل المشروع لأول مرة
افتح VS Code وافتح الترمنال:
```bash
# 1. اذهب للمجلد الذي تريد وضع المشروع فيه
# مثال في ويندوز:
cd C:\Users\اسم المستخدم\Desktop

# 2. تنزيل المشروع
git clone https://github.com/hoz2syr/BWP401.git

# 3. ادخل مجلد المشروع
cd BWP401

# 4. افتح المشروع في VS Code
code .
```

✅ المشروع الان على جهازك وجاهز!

---

### 🔹 الخطوة 2: قبل البدء بالعمل **كل يوم**
```bash
# 🔴 هذا الأمر الأهم - لا تنساه أبداً!
git checkout main
git pull origin main
```
✅ هذا يضمن أن لديك أحدث نسخة من عمل باقي الفريق.

---

### 🔹 الخطوة 3: إنشاء فرع خاص بمهمتك
```bash
# أنشئ فرع جديد وانتقل إليه مباشرة
git checkout -b feature/اسم-المهمة

# مثال:
# git checkout -b feature/login-page
# git checkout -b fix/price-calculation
# git checkout -b docs/installation-guide
```
✅ الآن كل التغييرات التي تقوم بها ستكون في فرعك الخاص ولن تؤثر على باقي الفريق.

---

### 🔹 الخطوة 4: ابدأ بالعمل والتعديل
✅ اكتب الكود، عدل الملفات، أنجز مهمتك كما تريد.

---

### 🔹 الخطوة 5: حفظ التغييرات
عند الانتهاء من جزء من العمل:
```bash
# 1. عرض الملفات التي عدلتها
git status

# 2. إضافة الملفات للحفظ
git add .

# 3. كتابة وصف واضح لما قمت به
git commit -m "feature: اضافة نظام تسجيل الدخول"
```

✅ اكتب رسائل واضحة يفهمها باقي الفريق!

---

### 🔹 الخطوة 6: قبل الرفع - خطوة مهمة جداً
```bash
# جلب آخر التغييرات مرة أخرى لتجنب المشاكل
git pull origin main
```
⚠️ إذا ظهرت تعارضات (conflicts) قم بحلها مع الفريق قبل المتابعة.

---

### 🔹 الخطوة 7: رفع فرعك إلى GitHub
```bash
# رفع الفرع الخاص بك إلى المستودع
git push origin feature/اسم-المهمة

# مثال:
# git push origin feature/login-page
```

---

### 🔹 الخطوة 8: إنشاء طلب الدمج (Pull Request)
1. اذهب إلى صفحة المشروع على GitHub
2. سوف ترى زر أخضر **"Compare & pull request"** اضغط عليه
3. أكمل النموذج:
   - اكتب عنوان واضح للمهمة
   - وصف تفصيلي لما تم إنجازه
   - اضف باقي أعضاء الفريق كمراجعين
4. اضغط على **"Create pull request"**

---

### 🔹 الخطوة 9: انتظر الموافقة
✅ لا تقم بدمج طلبك بنفسك!
❌ انتظر موافقة على الأقل عضو واحد آخر من الفريق

إذا وافق أحد الفريق يقوم قائد الفريق بالدمج إلى الفرع الرئيسي.

---

### 🔹 الخطوة 10: بعد الدمج بنجاح
```bash
# العودة للفرع الرئيسي
git checkout main

# جلب التحديثات الجديدة
git pull origin main

# حذف الفرع القديم (اختياري)
git branch -d feature/اسم-المهمة
```

✅ انتهيت! يمكنك الآن البدء بمهمة جديدة.

---

## 📌 ملخص سريع للأوامر بالترتيب
| الخطوة | الأمر |
|--------|-------|
| قبل البدء | `git checkout main && git pull` |
| إنشاء فرع | `git checkout -b feature/your-task` |
| بعد التعديل | `git add . && git commit -m "رسالة"` |
| قبل الرفع | `git pull origin main` |
| رفع الفرع | `git push origin feature/your-task` |
| بعد الدمج | `git checkout main && git pull` |

---

## ❗ أهم الملاحظات:
1. **لا تعمل أبداً مباشرة على الفرع `main`**
2. كل مهمة في فرع منفصل
3. لا تضع عمل يوم كامل في commit واحد - قسمه لقطع صغيرة
4. `git pull origin main` قبل البدء وقبل الرفع دائماً
5. تواصل مع الفريق قبل البدء بمهمة كبيرة

✅ الآن كل عضو في الفريق يعرف بالضبط ماذا يفعل خطوة بخطوة!
